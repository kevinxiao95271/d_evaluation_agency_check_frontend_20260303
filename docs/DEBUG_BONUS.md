# 附加项管理调试指南

## 如何验证 API 是否真的被调用

### 方法1: 浏览器控制台（推荐）

1. 打开浏览器访问 http://localhost:5039
2. 按 F12 打开开发者工具
3. 切换到 "Console" (控制台) 标签
4. 进入 "管理后台" → "附加项管理"
5. 切换任意机构的开关

**应该看到的日志**:
```
🔵 调用 bonusApi.set: {taskId: 1, institutionId: 1, directorPresentation: 1, secretaryParticipation: 0}
✅ API 返回结果: {id: 1, taskId: 1, institutionId: 1, ...}
```

### 方法2: 网络请求（最准确）

1. 打开浏览器访问 http://localhost:5039
2. 按 F12 打开开发者工具
3. 切换到 "Network" (网络) 标签
4. 进入 "管理后台" → "附加项管理"
5. 切换任意机构的开关

**应该看到的请求**:
- 请求: `POST http://localhost:5031/api/bonus`
- 状态: `200 OK`
- 响应: JSON 数据

### 方法3: 后端日志

查看后端控制台，应该能看到 POST /api/bonus 的请求日志

## 如果没有看到 API 调用

### 可能原因1: 浏览器缓存
**解决**: 强制刷新页面 (Ctrl+Shift+R 或 Ctrl+F5)

### 可能原因2: 代码未更新
**解决**: 
1. 检查开发服务器是否运行
2. 查看终端是否有 "hmr update" 提示
3. 重启开发服务器

### 可能原因3: API 代理问题
**检查**: 
1. 后端是否在 http://localhost:5031 运行
2. vite.config.js 中的代理配置是否正确

## 测试步骤

### 测试1: 单个开关切换
1. 找到任意机构
2. 点击"主任演讲"开关
3. 查看控制台日志
4. 查看网络请求
5. 应该显示"更新成功"提示

### 测试2: 编辑保存
1. 点击任意机构的"编辑"按钮
2. 修改附加项
3. 点击"确定"
4. 查看控制台日志
5. 应该显示"保存成功"提示

### 测试3: 批量操作
1. 点击"批量设置主任演讲"
2. 确认操作
3. 查看控制台日志（应该有多条）
4. 查看网络请求（应该有64个请求）

## 验证数据是否真的保存

### 方法1: 刷新页面
**问题**: 刷新后数据会丢失（显示为0）
**原因**: 后端缺少列表接口 GET /api/bonus/list

### 方法2: 直接查询后端
```bash
# 查询某个机构的附加项
curl http://localhost:5031/api/bonus/1/1
```

### 方法3: 使用测试页面
打开 `test_bonus_frontend.html` 测试 API

## 当前已知问题

1. **页面刷新后数据丢失**
   - 原因: 缺少 GET /api/bonus/list 接口
   - 影响: 无法加载已保存的数据
   - 解决: 需要后端实现列表接口

2. **批量操作较慢**
   - 原因: 逐个调用 API
   - 影响: 64个机构需要10-20秒
   - 解决: 需要后端实现批量接口

## 如果确认 API 被调用但数据不生效

可能是后端问题，需要检查：
1. 后端是否真的保存到数据库
2. 数据库连接是否正常
3. 后端日志是否有错误
