# 附加项管理测试指南

## 访问地址
http://localhost:5039

## 测试步骤

### 1. 进入附加项管理页面
1. 访问 http://localhost:5039
2. 登录系统（如需要）
3. 进入"管理后台" → "附加项管理"

### 2. 查看机构列表
- ✅ 应该显示所有机构（约64个）
- ✅ 默认选中第一个任务
- ✅ 所有机构的附加项默认为0

### 3. 测试单个更新
1. 找到任意机构
2. 切换"主任演讲"开关 → 应该显示 +8分
3. 切换"秘书参与"开关 → 应该显示 +2分
4. 同时开启两个 → 应该显示 +10分
5. 查看控制台，应该有成功的 API 调用

### 4. 测试编辑功能
1. 点击任意机构的"编辑"按钮
2. 在对话框中修改附加项
3. 点击"确定"保存
4. 应该显示"保存成功"提示

### 5. 测试批量操作
1. 点击"批量设置主任演讲" → 所有机构主任演讲变为"是"
2. 点击"批量设置秘书参与" → 所有机构秘书参与变为"是"
3. 点击"批量清空" → 所有机构附加项清空为0

注意：批量操作会逐个调用 API，64个机构约需10-20秒

### 6. 切换任务
1. 在顶部下拉框选择不同的任务
2. 应该重新加载该任务的附加项数据

## 已移除功能
- ❌ 历史记录按钮（已移除）

## 当前限制

### ⚠️ 列表数据
- 页面加载时显示默认值（全部为0）
- 原因：后端缺少 `GET /api/bonus/list?taskId={taskId}` 接口
- 影响：看不到之前设置的附加项数据
- 解决：需要后端实现列表接口

### ⚠️ 批量操作性能
- 批量操作较慢（64个机构约需10-20秒）
- 原因：逐个调用 API
- 影响：用户体验不佳
- 解决：需要后端实现 `POST /api/bonus/batch` 接口

## 验证要点

### ✅ 应该正常工作
- 加载任务列表
- 加载机构列表
- 单个开关切换并保存
- 编辑对话框保存
- 批量操作（虽然慢但能用）
- 附加分计算正确（主任8分 + 秘书2分）

### ⚠️ 已知问题
- 刷新页面后附加项数据丢失（显示为0）
- 批量操作较慢

## 后端 API 调用

### 成功的调用
```
POST http://localhost:5031/api/bonus
{
  "taskId": 1,
  "institutionId": 1,
  "directorPresentation": 1,
  "secretaryParticipation": 0
}
```

### 响应示例
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "taskId": 1,
    "institutionId": 1,
    "directorPresentation": 1,
    "secretaryParticipation": 0,
    "filledBy": null,
    "institutionName": "临床检验中心",
    "taskName": "2025年度考核"
  }
}
```
